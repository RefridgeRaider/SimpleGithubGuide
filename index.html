Getting Started

What are Git and GitHub?

Git

Git is a version control system that tracks changes in your code. It helps you save progress, compare versions, and collaborate safely.

GitHub

GitHub is a cloud platform that hosts Git repositories online. It helps teams share projects, review code, and manage collaboration.

Git Core Concepts

Understand the Basics of Git

Repository

A project folder tracked by Git.

Commit

A saved snapshot of your project at a certain moment.

Branch

A separate line of development used for features or fixes.

Why use Git?

Git makes it easy to track changes, undo mistakes, experiment safely, and work with others without overwriting each other’s code.

GitHub Features

What GitHub Helps You Do

Store Repositories Online

Keep your projects in the cloud so they are accessible from anywhere.

Collaborate with Teams

Invite others, review pull requests, and work together efficiently.

Track Issues

Organize bugs, enhancements, and tasks with GitHub Issues.

Showcase Projects

Build a portfolio and share your work publicly or privately.

Standard Workflow

A Simple Git & GitHub Workflow

1

Create or Clone a Repository

Start a new project or copy an existing repository to your computer.

2

Make Changes

Edit files, add features, or fix bugs in your local project.

3

Stage and Commit

Stage your changes and save them as a commit with a clear message.

4

Push to GitHub

Upload your commits from your computer to GitHub.

5

Open a Pull Request

Ask for review before merging your changes into the main branch.

Useful Commands

Common Git Commands

git init

Initialize a new Git repository.

git clone URL

Copy a repository from GitHub to your computer.

git status

Check the current state of your files.

git add .

Stage all changed files.

git commit -m "message"

Save staged changes with a commit message.

git push

Send your commits to GitHub.

git pull

Download the latest changes from GitHub.

git branch

View or manage branches.
